# Plan 04-02: Repo-Read Consent Gate And Shared Policy

<objective>
As a teammate running Claude from Codex, I want a clear allow-once,
repo-allow, or cancel choice before Claude Code reads this repo so that live
reviews are deliberate, inspectable, and revocable.
</objective>

<must_haves>
- D-31: Before live Claude review or rescue, the user sees that Claude Code may
  read this repo's diff, related files, and selected planning docs.
- D-32: Permission choices are allow once, always allow for this repository, and
  cancel.
- D-33: Repo-level consent can be inspected and revoked without hand-editing
  state files.
- D-34: Review, adversarial review, rescue, and future follow-up paths share the
  same consent gate.
- D-35: Repo-read consent does not imply `allow_write`.
</must_haves>

## Artifacts this plan produces

- `StateStore` consent methods for reading, persisting, and revoking repo-level
  repo-read consent
- Optional `repo_read_consent` schema on live Claude-launching tools
- Shared consent gate helper in `server.mjs`
- New MCP tools for consent inspection and revocation, such as
  `claude_consent_status` and `claude_consent_revoke`
- Skill updates that ask the allow-once / repo-allow / cancel question before
  calling live review tools
- Tests proving no Claude process starts without consent
- Docs explaining repo-read consent and separate write boundary
