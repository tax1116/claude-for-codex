# Plan 04-01: Codex Skill Surface And Compatibility Aliases

<objective>
As a Codex-first teammate, I want `$claude-review`, `$claude-adversarial`,
`$claude-rescue`, and `$claude-setup` skills to be the standard workflow surface
so that I can use Claude review without exposing long slash prompt internals in
the chat transcript.
</objective>

<must_haves>
- D-26: Codex skills become the standard team-facing workflow surface.
- D-27: Initial skill entry points are `$claude-review`,
  `$claude-adversarial`, `$claude-rescue`, and `$claude-setup`.
- D-28: Skills are thin launchers. MCP tools remain the source of truth for
  review policy, output format, safety, consent, and failure guidance.
- D-29: Slash prompts are optional compatibility aliases only.
- D-30: Keep `claude_adversarial_review`; do not add `$claude-design-review`.
- D-36: Docs distinguish product milestone labels from npm package version
  `1.0.0`.
</must_haves>
