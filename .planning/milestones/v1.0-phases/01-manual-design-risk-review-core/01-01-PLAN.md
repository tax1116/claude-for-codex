# Plan 01-01: Runtime Review Contract

<objective>
As a Codex user, I want `/claude-review` and `/claude-adversarial` to call a
read-only Claude Code review path with explicit context, actionable setup
diagnostics, prioritized findings, and no misleading edit or chat-transfer
claims, so that I can safely ask for a second opinion before committing work.
</objective>

<must_haves>
- D-01: Runtime must keep two review modes: implementation-risk review and adversarial design critique.
- D-02: Commands must work without required inputs; `base` and `focus` remain optional narrowing controls.
- D-03: Output must prioritize concrete findings over long narrative critique.
- D-05: Claude context must be explicit and must not imply full Codex chat transfer.
- D-06: Prompts must require `git status --short --branch` and relevant untracked file reads.
- D-07: Planning context must be current-phase scoped: `.planning/PROJECT.md`, `.planning/REQUIREMENTS.md`, `.planning/ROADMAP.md`, this phase `01-CONTEXT.md`, and phase plans when available.
- D-08: `resume` must mean Claude Code session continuity only.
- D-09: Review tools must stay read-only with `Read`, `Grep`, `Glob`, and read-style git commands.
- D-11: Results must explicitly state the review was read-only and no files were edited.
- D-13: Setup output must diagnose binary, auth/reachability, and timeout-prone config.
- D-14: Findings must use `High`, `Medium`, and `Low`.
- D-15: Clean output must say `No high-confidence findings` and include reviewed scope plus caveats.
- D-16: Failures must map to actionable categories where possible.
</must_haves>
