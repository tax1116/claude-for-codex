# Plan 02-02: Package And Background Workflow Documentation

<objective>
As a teammate installing the package, I want docs and package checks to show
the complete background review workflow so that I can start, observe, fetch,
and cancel long reviews without guessing which files or commands are required.
</objective>

<must_haves>
- D-11: Docs include examples for design review and adversarial review with focus.
- D-12: Docs include examples for review against a base ref.
- D-13: Docs include examples for background review, status, result, and cancel.
- D-14: Docs explain cancellation's process-lifetime limit.
- D-15: CI continues to run lint, syntax checks, tests, and npm package dry-run.
- D-16: Package dry-run includes every runtime file required by npm users.
</must_haves>

## Artifacts this plan produces

- README and setup examples for the full background lifecycle.
- Design docs explaining process-lifetime cancellation semantics.
- Docs/package tests that lock examples and package contents.
- Package contract updates if new runtime files were added by Plan 02-01.
